手把手教你用hugo制作一个个人的静态网页

​ 首先我们得通过网站了解一哈hugo到底是个啥？是干什么的？能吃吗？等你了解完了hugo强大的功能后，我想你也会说：真香！

​ 了解后我们可以更好的使用hugo：

快速安装

1、二进制安装（推荐：快速、简单）

到hugo Releases下载对应的操作系统的hugo二进制文件。

Mac下直接可以通过homebrew程序管理包的安装：

​ brew install hugo

2、源码安装（不推荐：麻烦、繁琐）

具体可以看看hugo中文文档，再次不描述。大致是通过安装Go顺带搞定hugo的安装。

生成站点

1、pwd确认当前路径，然后cd到你想生成站点的位子，比如：

cd C:\Users\Administrator\Desktop\123

直接进入123目录，敲到一下代码，hugo就能快速生成站点。

'hugo new site [新建目录名]'

之后可以cd到新建目录敲打tree指令展示站点目录结构

archetypes/

content/

layouts/

static/

config.toml

创建文章

创建一个about页面

hugo new about.md

about.md自动生生到了content/about.md,我们可以打开about.md看一哈：

+++

date = "2015-10-25T08:36:54-07:00" draft = true title = "about"

+++

用mackdown的格式写下你要记录的东西

内容都是markdown格式，+++之间的内容是TOML格式的，更具个人喜好，可以换成YAML格式或则JSON格式。

创造第一篇文章，放到content/psot目录，方便之后生成聚合页面。

hugo new post/first.md

vim first.md编辑或则使用其他方式打开编写.

皮肤安装

到皮肤列表选择一个皮肤，找到相应的*github*地址，再你自己的*theme*（新版本hugo site 的时候会创建*themes*）cd 到themes 里面 git clone

cd themes

git clone + 皮肤github

运行Hugo

再你的站点根目录执行*hugo*命令进行调试：

hugo server --theme=主题 --buildDrafts

浏览器里面打开http://localhist:1313

部署

假设你在GitHub Pages上部署，首先你要在*GitHub*上面创建一个*Repository，命名方式:自定义.github.io*(如：looklookface.github.io)。

在站点的根目录执行hugo命令生成最终页面：

hugo --theme-主题 --baseUrl=”http://looklookface.github.io“

（keep one’s eyes open ：以上命令并不会生成草稿页面，如果未生成任何文章，可以去掉文章头部的*draft=true*重新生成）在执行*hugo*命令后会在站点根目录生成一个public目录

如果一些都ok 现

就把你生成的*public*文件*push*到你创建的Repository里面的master分支呗。

cd public

git init

git remote add origin +自己的github url

git add .

git commit -m '描述'

git push -u origin master

最后就该去访问你的*GitHub*了：http://looklookface.github.io
